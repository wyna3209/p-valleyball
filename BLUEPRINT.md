# 10단계. 서버 게임 루프 만들기

## 목표

서버가 일정 간격으로 게임 상태를 계산합니다.

## 체크리스트

```text
□ setInterval로 서버 게임 루프 만들기
□ 초당 30회 또는 60회 업데이트 설정
□ 플레이어 입력을 바탕으로 위치 계산
□ 중력 적용
□ 바닥 충돌 처리
□ 화면 밖 이동 제한
□ 공 이동 처리
□ 공 중력 적용
□ 벽 충돌 처리
□ 게임 상태를 클라이언트에 전송
```

처음 추천:

```text
초당 30회
1000 / 30 = 약 33ms
```

즉:

```js
setInterval(updateGame, 1000 / 30);
```

처음부터 60fps로 욕심내지 않아도 됩니다.

> ⚠️ **delta time 기반 처리 권장**
>
> `setInterval`은 Node.js에서 타이밍이 정확하지 않아 누적 오차가 생길 수 있습니다.
> 물리 계산에 delta time을 곱해 실제 경과 시간 기반으로 처리하는 게 안전합니다.
>
> ```js
> let lastTime = Date.now();
>
> function updateGame() {
>   const now = Date.now();
>   const dt = (now - lastTime) / (1000 / 30); // 1프레임 기준 정규화
>   lastTime = now;
>
>   // 위치 계산 시 dt를 곱해서 사용
>   player.x += player.vx * dt;
> }
>
> setInterval(updateGame, 1000 / 30);
> ```

성공 기준:

```text
서버에서 계산한 위치가 양쪽 화면에 동일하게 보인다.
```

# 11단계. 플레이어 이동 구현

## 체크리스트

```text
□ 왼쪽 입력 시 vx 음수
□ 오른쪽 입력 시 vx 양수
□ 입력 없으면 vx 0
□ 점프 입력 시 바닥에 있을 때만 vy 음수
□ 매 프레임 x += vx
□ 매 프레임 y += vy
□ vy에 중력 추가
□ 바닥에 닿으면 y 고정
□ 바닥에 닿으면 vy = 0
□ 화면 밖으로 못 나가게 제한
□ 자기 진영 밖으로 못 넘어가게 제한 (필수)
```

> ✅ **진영 제한은 필수입니다.** 피카츄배구의 핵심 규칙이므로 반드시 구현합니다.
> 플레이어는 네트를 넘지 못합니다 (player1: x ≤ 네트위치, player2: x ≥ 네트위치).

피카츄배구식이면 보통 중앙 네트를 기준으로 진영을 나눕니다.

```text
player1: 왼쪽 진영
player2: 오른쪽 진영
```

성공 기준:

```text
각자 자기 캐릭터를 좌우 이동하고 점프할 수 있다.
```

# 12단계. 공 이동 구현

## 체크리스트

```text
□ 공 x 좌표 업데이트
□ 공 y 좌표 업데이트
□ 공 vx 적용
□ 공 vy 적용
□ 공에 중력 적용
□ 좌우 벽 충돌 처리
□ 천장 충돌 처리
□ 바닥 충돌 처리
□ 네트 충돌 처리 (필수)
```

> ✅ **네트 충돌은 필수입니다.** 공이 네트 하단부에 닿으면 튕겨야 합니다.
> 공이 네트 높이 이하로 이동할 때 x가 네트 범위 안이면 vx를 반전합니다.
> 플레이어도 네트를 통과하지 못합니다 (11단계 진영 제한과 연동).

초기 공 물리:

```text
x += vx
y += vy
vy += gravity
```

벽 충돌:

```text
왼쪽 벽에 닿으면 vx 반전
오른쪽 벽에 닿으면 vx 반전
천장에 닿으면 vy 반전
```

바닥 충돌:

```text
왼쪽 바닥에 공이 닿음 → player2 점수 +1
오른쪽 바닥에 공이 닿음 → player1 점수 +1
공 위치 리셋
```

성공 기준:

```text
공이 화면 안에서 움직이고 벽에 튕긴다.
```

# 13단계. 공과 플레이어 충돌 처리

## 체크리스트

```text
□ 공과 player1 거리 계산
□ 공과 player2 거리 계산
□ 일정 거리 이하이면 충돌로 판단
□ 충돌 시 공의 방향 변경
□ 플레이어가 움직이는 방향에 따라 공 속도 일부 반영
□ 점프 중 충돌하면 공이 더 위로 튀게 처리
□ 연속 충돌 방지용 쿨타임 적용
```

처음에는 단순 거리 충돌로 충분합니다.

```text
공 중심과 플레이어 중심의 거리가 일정 값보다 작으면 충돌
```

피카츄배구 느낌을 내려면:

```text
플레이어 머리 위쪽에 맞으면 위로 튐
왼쪽에서 맞으면 오른쪽으로 튐
오른쪽에서 맞으면 왼쪽으로 튐
```

성공 기준:

```text
플레이어가 공에 닿으면 공이 튕긴다.
```

# 14단계. 점수 처리

## 체크리스트

```text
□ 공이 왼쪽 바닥에 닿으면 player2 득점
□ 공이 오른쪽 바닥에 닿으면 player1 득점
□ 득점 후 공 위치 초기화
□ 플레이어 위치 초기화 여부 결정
□ 점수판 업데이트
□ 목표 점수 도달 시 게임 종료
```

예시 규칙:

```text
5점 먼저 얻으면 승리
```

성공 기준:

```text
공이 바닥에 떨어지면 점수가 오른다.
양쪽 화면에 같은 점수가 보인다.
```

# 15단계. gameState 전송

## 목표

서버가 계산한 최신 상태를 같은 방의 모든 클라이언트에게 보냅니다.

## 체크리스트

```text
□ 서버 루프마다 gameState 이벤트 보내기
□ roomId 기준으로 같은 방에만 전송
□ players 위치 전송
□ ball 위치 전송
□ score 전송
□ gameStatus 전송
```

전송 데이터 예시:

```js
{
  players: {
    p1: { x: 150, y: 400 },
    p2: { x: 650, y: 400 }
  },
  ball: { x: 400, y: 200 },
  score: { p1: 0, p2: 0 },
  status: "playing"
}
```

성공 기준:

```text
player1 화면과 player2 화면의 공 위치, 캐릭터 위치, 점수가 같다.
```

# 16단계. 프론트에서 gameState 반영

## 체크리스트

```text
□ socket.on("gameState") 작성
□ 받은 player1 좌표를 Phaser 객체에 반영
□ 받은 player2 좌표를 Phaser 객체에 반영
□ 받은 ball 좌표를 Phaser 객체에 반영
□ 받은 score를 텍스트에 반영
□ status가 waiting이면 대기 화면 표시
□ status가 playing이면 게임 화면 표시
□ status가 ended이면 결과 화면 표시
```

중요:

```text
프론트에서 공 위치를 직접 계산하지 않기
프론트에서는 받은 좌표를 표시만 하기
```

성공 기준:

```text
서버 상태가 바뀌면 화면도 그대로 바뀐다.
```

# 17단계. 대기 상태와 게임 시작

## 체크리스트

```text
□ player1만 들어오면 waiting 상태
□ player2가 들어오면 ready 상태
□ 둘 다 들어오면 자동 시작
□ 또는 player1이 시작 버튼 누르면 시작
□ playing 상태일 때만 게임 루프 계산
□ waiting 상태일 때는 공 정지
```

초기에는 자동 시작 추천:

```text
2명 입장 → 3초 카운트다운 → 게임 시작
```

성공 기준:

```text
한 명만 접속하면 대기
두 명이 접속하면 게임 시작
```
